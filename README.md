# Qt 摄像头实时采集与抓拍系统 (XDMA Stream 高性能版)

本项目基于 Qt 框架开发，专为工业级高速数据采集与图像处理场景设计。系统深度集成了 **XDMA AXI4-Stream** 传输机制，相比传统的 DMA 寄存器访问，能够提供更低的延迟和更高的吞吐量。

## 🚀 项目特性

* **高性能 XDMA Stream 传输**：
    * **自定义包大小**：支持根据硬件带宽与内存负载动态配置单次传输的数据包长度（Packet Size）。
    * **可调传输间隔**：支持自定义传输时间间隔，在实时性与系统功耗之间取得最佳平衡。
* **实时预览**：支持 30FPS 稳定流输出，分辨率锁定为 **640 * 360**。
* **硬件加速**：集成 `XDMA_MoreB` 驱动库，优化 PCIe 数据链路，大幅降低 CPU 占用。
* **格式处理**：直接支持摄像头/采集卡原始输出的 **YUY2** 像素格式。
* **一键抓拍**：支持实时画面无损截取并保存为本地图像。
* **跨编译器支持**：完美兼容 MSVC 2017+ 与 MinGW 编译器，源码采用 UTF-8 编码。

## 🛠 技术规格

| 参数项 | 设定值 | 说明 |
| :--- | :--- | :--- |
| **分辨率** | 640 * 360 | 默认工作分辨率 |
| **帧率 (FPS)** | 30 | 实时采集频率 |
| **像素格式** | YUY2 | 原始数据流格式 |
| **传输模式** | **XDMA Stream** | 基于 AXI4-Stream 的流式传输 |
| **包大小 (Packet)** | 可配置 | 根据具体硬件 IP 核设定 |
| **传输间隔** | 可自定义 | 优化实时响应速度 |
| **C++ 标准** | C++17 | 现代 C++ 特性支持 |

按钮分配：
打开XDMA通道并自检：仅负责 XDMA 打开与硬件自检。
发送XDMA链路测试包：仅负责硬件链路测试包发送（依赖 XDMA 已打开）。
运行协议封包自测：仅负责软件协议自测（不依赖 XDMA）。
发送缓存帧(封包+1MiB)：发送抓拍缓存帧（走封包+聚合链路）。
开始/停止实时视频发送(封包+1MiB)：仅控制实时视频发送开关。
